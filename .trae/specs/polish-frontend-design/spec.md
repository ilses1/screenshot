# 设置页视觉与交互细化 Spec

## Why
当前设置页已完成 React + antd 重构，但整体仍偏“默认组件堆叠”，缺少清晰的信息层级与更贴合“截图工具”的界面气质。通过前端设计规范化与少量交互细化，可以显著提升观感与可用性，同时不改动任何 IPC 接口。

## What Changes
- 为 `index`（设置页/历史页）确立统一的视觉方向与基础样式（字号、间距、背景、分隔、对比度），并在实现中保持克制与一致性。
- 在不新增 IPC 能力、不更改 `window.api` 接口的前提下，优化以下界面与交互：
  - 页面结构：标题区、设置区、历史区的层级更清晰；在常见窗口尺寸下布局更紧凑且不拥挤。
  - 表单体验：更合理的字段分组、标签文案、禁用态/加载态；保存与重新加载的反馈更明确。
  - 历史列表：列宽与文本溢出策略更合理；空状态、加载态更友好；复制路径交互更明显。
- 仅对 `index` 页面生效；`capture` / `editor` 两个入口不做 UI 变更。
- **不新增依赖**：优先使用现有 `antd` / `dayjs` / React 能力完成。

## Impact
- Affected specs: 渲染进程设置页的视觉规范与交互细化
- Affected code: `src/renderer/main.tsx`（设置页与历史列表 UI）、可能新增 `src/renderer/index.css`（全局样式，仅影响 index）

## ADDED Requirements
### Requirement: 视觉方向
系统 SHALL 采用“工业化工具面板（Graphite + Accent）”的视觉方向：
- 背景：深色石墨/蓝黑基底，卡片分层更明确
- 主强调色：用于主按钮与关键操作状态（保持 antd 主题一致）
- 字体与排版：标题更克制、正文更紧凑、信息层级清晰

#### Scenario: 初次打开设置页
- **WHEN** 用户打开设置页
- **THEN** 页面呈现清晰的标题区与两块主卡片（设置/历史），整体视觉统一且对比度达标

### Requirement: 交互反馈一致性
系统 SHALL 对关键操作提供一致且可预期的反馈：
- 加载设置/加载历史：在对应卡片/列表展示加载态
- 保存设置：成功提示与按钮 loading 一致；校验失败不弹“保存失败”
- 清空历史：保持二次确认；成功后立即刷新列表

#### Scenario: 保存设置（成功）
- **WHEN** 用户点击保存且通过校验
- **THEN** 按钮进入 loading，保存成功后展示成功提示并退出 loading

#### Scenario: 保存设置（校验失败）
- **WHEN** 用户点击保存但表单校验失败
- **THEN** 仅展示字段错误，不展示“保存失败”类全局错误提示

## MODIFIED Requirements
### Requirement: 设置页功能不变
现有能力（读写设置、刷新/清空历史、贴最近截图） SHALL 保持行为一致；不得修改 `window.api` 方法签名或增加新方法。

## REMOVED Requirements
无
