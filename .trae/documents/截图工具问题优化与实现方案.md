## 需求梳理（根据 记录问题.md ）
- 功能1：全局截图快捷键设置为 F2，避免与常见系统/应用快捷键冲突，可允许用户自定义。
- 功能2：应用启动后最小化到系统托盘（右下角通知区域），使用清晰可识别的应用图标，并提供右键菜单。
- 功能3：截图完成后进入编辑模式，支持添加文字、绘制矩形、高亮、箭头等基础标注功能，并可方便保存/复制。

- 初期只做核心闭环：截图 → 编辑 → 保存/复制，后续再扩展更多样式/撤销历史等高级功能。

## 功能1：快捷键 F2 配置与行为
- 目标
  - 支持在应用设置中配置全局截图快捷键，默认 F2。
  - 启动时自动注册全局快捷键，失败时给出提示（如被占用）。
- 行为细化
  - 用户在任意窗口按下 F2：
    - 当前窗口不失焦（或尽量少打扰）。
    - 进入全屏/区域截图模式（视现有实现而定）。
  - 设置界面：
    - 显示当前快捷键。
    - 提供“点击更改快捷键”交互，录制新的组合键（如 Ctrl+Shift+F2）。
    - 检查与 Windows 保留快捷键和常用系统快捷键的冲突，必要时给出警告。
- 技术注意点（后续实现时参考）
  - 使用系统级全局热键注册机制（如 Win32 RegisterHotKey / 对应跨平台库）。
  - 存储到配置文件或注册表，保证重启后生效。

## 功能2：系统托盘图标与行为
- 目标
  - 应用启动后不占用任务栏空间，常驻系统托盘。
  - 托盘图标清晰反映应用状态，并提供常用操作入口。
- 行为细化
  - 启动应用：
    - 主窗口可选择是否显示一次（通过设置控制，如“开机启动时不显示主窗口”）。
    - 最终收纳到托盘，只保留托盘图标。
  - 托盘图标：
    - 使用自定义图标（.ico），在浅色/深色主题下清晰可见。
    - 鼠标悬停显示提示文字，如“截图助手 - 按 F2 开始截图”。
  - 托盘右键菜单建议项：
    - “开始截图”（等同于 F2）。
    - “打开主界面/设置”。
    - “开机自启动”勾选项（如有需求）。
    - “退出”。
- 技术注意点（后续实现时参考）
  - 确保应用退出时正确释放托盘图标，避免“僵尸图标”。
  - 考虑多实例启动时的行为：只保留一个实例和一个托盘图标。

## 功能3：截图后编辑功能设计
- 目标
  - 截图完成后自动进入编辑画布，让用户能快速做标注，并一键保存或复制到剪贴板。
- 基本编辑工具
  - 文字工具：
    - 点击画布添加文本框，可输入文字。
    - 支持字体大小、颜色简单设置（至少颜色、大小两项）。
  - 矩形/高亮框：
    - 绘制实线或半透明填充矩形，用于框选重点区域。
    - 提供边框颜色/填充颜色预设。
  - 箭头工具：
    - 起点到终点绘制箭头，用于指示重点。
    - 支持线条颜色与粗细。
- 通用编辑能力
  - 选中/移动：
    - 支持选中已有标注元素，拖动调整位置。
  - 删除：
    - 选中后按 Delete 删除，或提供小型“垃圾桶/删除”按钮。
  - 撤销/重做（可作为第二阶段）：
    - 提供最基本的 Ctrl+Z / Ctrl+Y 或按钮操作。
- 导出与关闭流程
  - 导出方式：
    - “复制到剪贴板”：复制当前编辑后的图像，方便直接粘贴到聊天/文档。
    - “保存到文件”：弹出文件保存对话框，支持 PNG（优先）和 JPG。
  - 关闭编辑器：
    - 若有未保存修改，可提示“是否保存/复制再关闭”。

## 交互流程示例
- 流程 1：用户使用快捷键截图
  - 在任意窗口按 F2 → 进入截图模式 → 用户框选区域 → 截图完成后自动打开编辑器窗口 → 用户添加箭头和文字 → 点击“复制到剪贴板” → 编辑器自动关闭（可由设置决定是否自动关闭）。
- 流程 2：从托盘发起截图
  - 右键托盘图标 → 选择“开始截图” → 后续流程与 F2 相同。

## 优先级与迭代建议
- 第一阶段（核心闭环）
  - 实现全局快捷键（默认 F2）触发截图。
  - 实现托盘图标与基本右键菜单（开始截图 / 退出）。
  - 实现截图后编辑器的最基础工具：矩形、箭头、文字 + 复制到剪贴板。
- 第二阶段（体验优化）
  - 完善快捷键自定义与冲突检测。
  - 增加颜色/粗细预设、撤销重做、简单历史记录。
  - 托盘菜单增加开机自启动、主界面/设置入口。
