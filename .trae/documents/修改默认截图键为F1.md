# 修改默认截图键为 F1（Plan）

## Summary
- 将截图工具默认全局截图快捷键从 `F2` 调整为 `F1`
- 不迁移、不覆盖现有用户配置：仅在“无配置 / 配置解析失败 / hotkey 为空”时使用 `F1` 作为默认值
- 同步更新设置页输入框提示文案示例为 `F1`

## Current State Analysis
- 主进程在 [index.ts](file:///c:/Users/wusan/Desktop/screenshot/src/main/index.ts) 中：
  - `loadConfig()` 默认 `hotkey` 为 `F2`（`parsed.hotkey ?? 'F2'`，以及无配置时 `hotkey: 'F2'`）
  - `registerShortcuts()` 在 `hotkey` 为空时回填 `F2` 并落盘
  - `createTray()` / `updateTrayMenu()` 在 tooltip 兜底文案里使用 `F2`
- 渲染进程设置页在 [main.ts](file:///c:/Users/wusan/Desktop/screenshot/src/renderer/main.ts) 中：
  - “截图快捷键”输入框 placeholder 示例为 `F2`

## Proposed Changes
### 1) 主进程：默认值改为 F1
- 文件：[src/main/index.ts](file:///c:/Users/wusan/Desktop/screenshot/src/main/index.ts)
- 修改点：
  - `loadConfig()`：
    - `parsed.hotkey ?? 'F2'` 改为 `parsed.hotkey ?? 'F1'`
    - 无配置/解析失败时的默认对象 `hotkey: 'F2'` 改为 `hotkey: 'F1'`
  - `registerShortcuts()`：
    - `hotkey` 为空时回填值从 `F2` 改为 `F1`（并保持 `saveConfig()` 行为不变）
  - `createTray()` / `updateTrayMenu()`：
    - tooltip 文案里 `config?.hotkey || 'F2'` 的兜底改为 `F1`
- 预期效果：
  - 新用户（无 `config.json`）启动后托盘提示与快捷键注册默认使用 `F1`
  - 已有 `config.json` 且 `hotkey` 有值的用户保持原有快捷键不变

### 2) 渲染进程：设置页提示文案同步
- 文件：[src/renderer/main.ts](file:///c:/Users/wusan/Desktop/screenshot/src/renderer/main.ts)
- 修改点：
  - “截图快捷键”输入框 placeholder 从 `例如 F2 或 Ctrl+Shift+F2` 改为 `例如 F1 或 Ctrl+Shift+F1`

## Assumptions & Decisions
- 决策：不做“迁移到 F1”的一次性写回；避免覆盖老用户显式配置（包括用户可能仍想用 `F2` 的场景）
- 假设：Electron `globalShortcut.register('F1', ...)` 在目标环境可用；如注册失败，现有兜底快捷键逻辑继续生效（`Alt+Shift+A`、`CommandOrControl+Shift+A`）

## Verification Steps
- 静态核对：
  - 全仓搜索 `F2`，确认默认值/提示文案相关位置已替换为 `F1`，且未误改非默认语义的文字
- 运行验证（开发模式）：
  - 以无 `config.json` 的用户数据启动一次，确认：
    - 托盘提示显示 `F1`
    - 按 `F1` 可进入截图；再次按 `F1` 可退出截图遮罩
  - 在设置页改为其他快捷键（如 `Ctrl+Shift+F3`），保存后确认热键生效且重启仍保持该值
