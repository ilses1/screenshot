## 当前项目模块拆分（按你仓库现状）
- 截图采集与选区（渲染页）：`desktopCapturer` 拿全屏图 → 透明置顶全屏窗口做选区遮罩 → 裁剪离屏 canvas → 写入剪贴板 → IPC 落盘
  - 代码位置：src/renderer/capture.ts、src/renderer/capture.html
- 标注编辑（渲染页）：canvas 叠加绘制（当前仅 pen/rect + undo）→ 复制到剪贴板/保存
  - 代码位置：src/renderer/editor.ts、src/renderer/editor.html
- 主进程编排：Tray 菜单、globalShortcut、窗口生命周期（capture/editor/pin）、配置与历史（JSON）、保存图片、贴图置顶
  - 代码位置：src/main/index.ts
- IPC 契约与类型：channel 常量、AppConfig/ScreenshotRecord
  - 代码位置：src/common/ipcChannels.ts、src/common/types.ts

## 落点建议（结合你当前代码，可直接开工的方向）

### 选区遮罩（capture）
- 交互状态机：把“拖拽选择”扩展为 idle / selecting / moving / resizing，支持选区移动与边角缩放（对标 Flameshot/ksnip）。
- 信息提示：实时展示宽高、坐标；鼠标附近显示十字准线；Esc 取消、Enter 确认、右键菜单（复制/保存/编辑/取消）。
- 放大镜：基于背景截图（或离屏 canvas）在光标附近做局部放大预览，提升精确度（对标系统截图工具与 Flameshot）。
- 键盘微调：方向键 1px，Shift+方向键 10px，保证可精确对齐（对标 ShareX/Greenshot 常见交互）。

### 截图采集（desktopCapturer）
- 多屏与鼠标屏幕：用 `screen.getCursorScreenPoint()` + `screen.getDisplayNearestPoint()` 选择当前屏幕源，避免永远取第一个屏。
- DPI/缩放：用 display.scaleFactor 校准缩放，确保裁剪区域与 thumbnail 像素坐标一致（高 DPI 下最常见的坑）。
- 避免“把遮罩窗口截进去”：推荐流程是先获取屏幕 thumbnail，再展示 captureWindow；或在采集前临时隐藏/透明化遮罩窗口，再立即恢复。

### 标注编辑（editor）
- 从 strokes 升级到“对象模型”：引入 Konva.js 或 Fabric.js，让图形具备选择、控制点、编辑属性（颜色/粗细/填充/透明度）和图层顺序。
- 工具集优先级：箭头、文字、马赛克/模糊、序号标记、取色器、撤销/重做（对标 Flameshot/ksnip/Greenshot）。
- 撤销栈：按“操作”而不是按“帧”记录（例如新增图形、移动、改颜色、裁剪），更接近专业截图工具体验。

### OCR（editor：占位按钮落地）
- 本地 OCR：用 tesseract.js worker 跑识别，避免阻塞 UI；提供进度、取消、语言选择（中/英）、结果复制。
- 结果体验：支持“只复制文字 / 复制文字+换行修正 / 复制为 Markdown”，并把结果写入历史记录（便于追溯）。

### 上传/工作流（form-data 依赖落地）
- 上传抽象：把“截图导出”做成管线：复制 → 保存 → 打开编辑 → OCR → 上传（对标 ShareX 的动作链）。
- 默认安全：上传必须显式点击确认；避免误上传敏感内容（Flameshot 社区也对默认上传较谨慎）。
- 目的地插件化：HTTP（multipart/form-data）、自定义脚本、企业 IM Webhook，统一成一个接口，便于扩展。

### 贴图置顶（pin）
- 多贴图：从“只 pin last”扩展为“可 pin 多张”，并提供关闭全部/关闭当前的快捷键。
- 窗口能力：可拖拽、缩放、透明度调节、锁定位置、跨屏移动（对标 ksnip 的 pinned images）。

### 许可提示
- Flameshot / ShareX / Greenshot / ksnip 多为 GPL 系：建议参考交互与架构思路，避免直接复制代码；若要直接复用代码/模块，优先选 MIT/Apache/BSD 许可证的库（例如 Konva/Fabric/tesseract.js）。
