# 基于 Electron 的桌面截图工具设计方案

## 一、目标与 MVP 范围
- 目标：做一个类似 Snipaste 的 Windows 桌面截图工具，基于 Electron，支持常驻托盘、全局快捷键截图和基础标注。
- MVP 范围（第一阶段先实现）：
  - 托盘常驻 + 主进程运行
  - 全局快捷键触发截图（如：Alt+` 或自定义）
  - 全屏截图 + 鼠标框选区域
  - 将选中区域复制到剪贴板，并可保存到本地文件
  - 简单设置界面（修改快捷键、保存路径等）

## 二、核心功能拆解

### 1. 托盘常驻与主窗口
- 使用 Electron 主进程创建 Tray 图标，提供菜单项：截图、设置、退出。
- 可选：提供一个主窗口（非必须），只用于设置与查看历史截图。

### 2. 全局快捷键
- 在主进程使用 `globalShortcut.register` 注册全局快捷键。
- 快捷键触发后：
  - 隐藏主窗口（如果有），防止被截进图片。
  - 创建“截图覆盖层窗口”（全屏透明无边框窗口）。

### 3. 截图与区域选择
- 使用 `desktopCapturer.getSources` 获取屏幕内容：
  - 选择整个屏幕源，获取视频流，在渲染进程里用 `<video> + <canvas>` 绘制成静态截图，或直接使用 `capturePage`（只截当前屏幕）作为备选。
- 渲染层实现一个“全屏覆盖层”：
  - 窗口属性：无边框、透明背景、总在最前、全屏。
  - 初始状态显示整屏截图图像，禁止与桌面交互（点击都落在覆盖层）。
  - 鼠标按下开始绘制选择框，移动时动态显示选区，松开后确定区域。
- 截图裁剪：
  - 根据选区坐标在 canvas 上裁剪对应区域生成新图像（PNG）。
  - 若用户按 ESC，取消截图，关闭覆盖层窗口。

### 4. 剪贴板与文件保存
- 使用 Electron 的 `clipboard` 模块：
  - 将裁剪后的图像写入剪贴板（`clipboard.writeImage`）。
- 文件保存：
  - 使用 `dialog.showSaveDialog` 或预设目录（如 `~/Pictures/Snip`）。
  - 通过主进程 `fs` 写入 PNG 文件。

### 5. 简单标注工具（第二阶段）
- 在裁剪完成后，打开一个“小编辑窗口”：
  - 加载截图图像。
  - 提供简单工具：矩形框、箭头、文字、画笔、马赛克（可按优先级逐步实现）。
- 编辑完成后：
  - 将结果再次写入剪贴板 / 保存文件。

### 6. 历史记录与管理（后续阶段）
- 本地维护截图历史：
  - 保存路径 + 元数据（时间、尺寸、来源屏幕等）到一个 JSON/SQLite。
- 提供历史列表界面：
  - 缩略图 + 复制到剪贴板 + 打开文件位置 + 删除记录。

### 7. 设置与配置
- 设置项：
  - 快捷键配置
  - 默认保存目录
  - 是否自动复制到剪贴板
  - 是否每次截图后打开编辑器
- 使用 `electron-store` 或自己封装 JSON 配置文件保存设置。

## 三、技术选型与整体架构

### 1. 技术选型
- 主框架：Electron（最新稳定版）。
- 语言：TypeScript（建议，便于管理主/渲染进程类型）。
- 渲染层：
  - 可选：React / Vue / 纯 TS + 原生 DOM。
  - 若功能会持续扩展（贴图、复杂设置等），推荐 React + Vite。

### 2. 进程架构
- 主进程：
  - 负责创建 Tray、管理 BrowserWindow（主窗口、截图覆盖层、编辑窗口）。
  - 注册全局快捷键、处理文件读写、配置读写。
  - 暴露安全 IPC 接口给渲染进程（通过 `ipcMain`）。
- 预加载脚本（preload）：
  - 使用 `contextBridge.exposeInMainWorld` 暴露有限 API。
  - 提供给渲染层的接口，例如：`captureScreen()`, `saveImage()`, `getSettings()` 等。
- 渲染进程：
  - UI 展示、用户交互、截图裁剪和标注逻辑。

### 3. 项目结构建议
- `/main`：主进程代码（入口 `main.ts`、窗口管理、Tray、globalShortcut、配置管理）。
- `/preload`：预加载脚本，桥接 API。
- `/renderer`：渲染层 UI（截图覆盖层页面、设置页面、历史记录页面、编辑器页面）。
- `/assets`：图标、Tray 图标等资源。
- `/config`：配置 schema、默认配置。

## 四、迭代步骤规划

### 阶段 1：项目初始化与基础架构
- 初始化 Electron + TypeScript +（可选）React/Vite 项目结构。
- 配置主进程、预加载脚本、安全上下文隔离。
- 实现 Tray 图标和基础菜单（截图、设置、退出）。

### 阶段 2：全局快捷键与基础截屏
- 主进程注册全局快捷键。
- 实现全屏覆盖层窗口：
  - 打开时自动获取当前屏幕截图并显示。
  - 支持鼠标拖拽选择区域，ESC 取消。
- 根据选区坐标裁剪图片，复制到剪贴板，保存到默认目录。

### 阶段 3：基础设置页面
- 创建设置窗口（或使用主窗口）：
  - 修改快捷键（需要在主进程重新注册）。
  - 修改默认保存目录。
  - 切换“截图后自动打开编辑器”等选项。
- 实现设置持久化（electron-store/自定义 JSON）。

### 阶段 4：标注编辑器
- 创建编辑窗口：
  - 加载截图图像到 canvas。
  - 实现矩形框、箭头、文本三个基础工具。
- 导出编辑结果到剪贴板/文件。

### 阶段 5：历史记录与增强
- 设计截图元数据结构并落地存储。
- 实现历史列表界面与基本操作。
- 逐步加入 Snipaste 式增强功能：
  - 贴图到桌面（将某张截图作为顶置小窗口固定显示）。
  - OCR（调用系统/第三方服务）。
  - 自动上传图床等（后期再考虑）。

## 五、关键实现点说明

### 1. 截图覆盖层的实现思路
- 窗口参数：`transparent: true`, `frame: false`, `fullscreen: true`, `alwaysOnTop: true`。
- 启动逻辑：
  - 主进程创建窗口，传入要显示的截图数据（通过 `BrowserWindow.webContents.send` 或在创建时注入 URL 参数）。
- 渲染层：
  - 显示原始全屏截图作为背景。
  - 使用一个半透明遮罩层 + 选区高亮区域制造“挖空”效果。
  - 使用鼠标事件记录起点/终点坐标并实时更新选区矩形。

### 2. 屏幕截图的数据流
- 主进程使用 `desktopCapturer` 或 `capturePage` 获取 `NativeImage`。
- 将 `NativeImage.toPNG()` 或 dataURL 传给渲染进程：
  - 方式一：通过 IPC 发送 Buffer。
  - 方式二：保存到临时文件，然后传路径给渲染层加载（更适合大图）。
- 渲染层根据用户选区裁剪并输出最终 PNG。

### 3. 安全与性能考虑
- 开启 `contextIsolation`，禁用 `nodeIntegration`，通过 preload 暴露有限 API。
- 截图与裁剪尽量在渲染层 canvas 内完成，减少主/渲染进程之间大数据拷贝。
- 对于多显示器场景：
  - 使用 `screen.getAllDisplays()`，决定按“当前鼠标所在屏幕”截图，或一次性支持所有屏幕（可后续增强）。

## 六、后续可选高级功能（对齐 Snipaste 特性）
- 贴图到桌面：
  - 将任意截图以无边框顶置小窗口显示，可拖动、缩放、保持在最前。
- 颜色拾取、像素坐标显示。
- 文本识别（OCR）。
- 自动上传到图床或团队图像服务器，返回链接。
