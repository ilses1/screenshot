## 目标
- 默认截图快捷键从 `Alt+\`` 改为 `F2`。
- 让“如何结束截图/取消截图”的操作对用户更明显、好理解。

## 修改点一：默认截图键改为 F2
- 现状：
  - 主进程中配置默认值是 `Alt+\``，并写入到用户配置文件后生效。
  - 设置页面只是读取/修改这个配置，不直接决定默认值。
- 计划改动：
  - 在主进程 `loadConfig` 中，将默认配置里的 `hotkey` 从 `Alt+\`` 改为 `F2`，只在以下情况使用：
    - 配置文件不存在；或
    - 配置文件存在但没有 `hotkey` 字段/字段为空。
  - 保持已经有配置文件的用户原来的快捷键不被强制改动，仅对新用户或未设置的情况使用 F2。
  - 设置页逻辑不改，只是读取/保存新的默认值，因此 UI 会自然显示 F2。

## 修改点二：结束截图的交互说明与补充
- 现状：
  - 当前逻辑是：按快捷键进入截图 → 按住鼠标左键拖动选择区域 → 松开鼠标左键即完成截图并关闭窗口；按 `Esc` 取消截图。
  - 界面上没有任何文字提示，容易让人不知道“怎么结束/取消”。
- 计划改动：
  - 在截图页面中增加显式文案提示，例如在屏幕左上角显示一行文字：
    - “按住左键拖动选择区域，松开完成截图，Esc 取消”。
    - 实现方式：在 `capture.html` 中加一个固定定位的文本元素，用简单样式（白字、半透明背景）渲染即可。
  - 在截图逻辑中保持现有完成方式不变（鼠标左键松开即完成）。
  - 补充一个更直观的取消方式：
    - 监听右键（`contextmenu` 或 `mousedown` 的右键），直接调用 `window.close()`，等价于取消截图。

## 验证步骤
- 启动开发环境：`npm run dev`。
- 看终端日志：
  - 确认主进程输出 `registerShortcuts F2 success`（或类似）说明 F2 注册成功。
- 在桌面按 F2：
  - 应该进入截图模式，看到鼠标变为十字，并在屏幕某处看到“如何结束截图/取消”的提示文字。
  - 按提示：
    - 按住左键拖动，松开后截图窗口关闭，在任意应用中 `Ctrl+V` 应可粘贴出截图。
    - 按 `Esc` 或右键，应取消截图，窗口关闭且不生成新截图记录。
