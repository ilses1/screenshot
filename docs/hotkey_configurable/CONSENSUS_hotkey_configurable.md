## 需求描述

在设置页允许用户修改“截图快捷键”。用户点击“修改”后进入录制态，按下新的按键组合后可确认应用；保存设置后，主进程持久化该快捷键并用其注册全局截图快捷键，托盘提示同步更新。

## 验收标准

- 设置页显示当前截图快捷键，支持进入录制态采集用户按键组合并回填显示。
- 点击“保存设置”后，快捷键变更会持久化到 `config.json`，重启应用仍保持。
- 快捷键注册成功后，按该快捷键可触发截图流程（与原 F1 行为一致）。
- 快捷键注册失败时：
  - 设置保存失败并提示用户（例如“快捷键被占用/不可用”）。
  - 配置保持原快捷键（不写入不可用的值），托盘提示不变化。
- 托盘 tooltip 文案显示当前快捷键（截图中/非截图中两种状态均正确）。

## 技术实现方案（约束与集成）

- 数据模型：复用 `AppConfig.hotkey` 字段；默认值 `F1`。
- 渲染进程：在设置页增加快捷键编辑 UI（录制态通过 `keydown` 监听采集组合，生成 accelerator 字符串）。
- 主进程：
  - `loadConfig()` 读取 `hotkey` 并校验；无效或缺失回落为 `F1`。
  - `registerShortcuts()` 使用 `config.hotkey` 注册，不再强制写回 `F1`。
  - 更新 IPC `SETTINGS_UPDATE` 允许传入 `hotkey`，并在注册失败时回滚。
- 兼容性：旧配置文件未包含/包含无效 hotkey 时自动回落到 `F1`。

## 边界限制

- 不实现跨应用的占用检测，仅在 `globalShortcut.register` 返回失败时提示。
- 不支持“空快捷键”；若用户尝试清空，保存时会提示并拒绝。
