# electron-screenshot

[English](README.md)

一个基于 Electron 的托盘截图工具：使用全局快捷键触发截图，拖拽框选区域，确认后写入剪贴板，并可按设置自动保存文件 / 自动打开编辑器。

## 特性

- 托盘常驻：支持一键“截图 / 设置 / 退出”
- 全局截图快捷键（默认 `F1`），可在设置页修改
- 框选交互：拖拽框选，点击 `✓` 或按 `Enter` 确认；按 `Esc` 或右键取消
- 确认后写入剪贴板
- 可选：自动保存 PNG 到文件，并在设置页查看本地历史（支持复制路径）
- 可选：截图后自动打开编辑器
- 可调遮罩透明度（`maskAlpha`）
- “贴最近截图”：将最近一次截图以置顶窗口形式贴出（可拖拽/缩放）

## 普通用户安装（从 Release 下载）

目前 Release 仅提供 Windows 安装包。

1. 打开 GitHub 仓库的 Releases 页面，进入最新版本。
2. 下载 `.exe` 安装包（通常名为 `Electron Screenshot Setup *.exe`）。
3. 双击安装（如遇 Windows SmartScreen 提示，选择“仍要运行”）。
4. 安装完成后应用会常驻托盘：
   - 默认按 `F1` 触发截图
   - 托盘菜单可进入“设置”修改快捷键 / 保存目录等

提示：
- 若全局快捷键注册失败：尝试更换快捷键组合，或以管理员权限运行。

## 安装依赖

前置要求：

- Node.js（建议使用最新 LTS）
- pnpm（见 `package.json#packageManager`）

安装：

```bash
pnpm install
```

## 运行命令

```bash
pnpm dev
pnpm build
pnpm start
pnpm dist
pnpm test
```

- `pnpm dev`：开发模式启动（renderer 走 Vite dev server）。
- `pnpm build`：构建产物到 `out/`（用于预览与打包）。
- `pnpm start`：预览构建产物（建议先执行 `pnpm build`）。
- `pnpm dist`：electron-builder 打包生成安装包（建议先执行 `pnpm build`）。
- `pnpm test`：运行 Vitest 测试。

## 配置项

托盘菜单 → “设置”。

- 截图快捷键（`hotkey`）
  - 默认：`F1`
  - 点击输入框进入录制态，按下新的按键组合后保存即可生效
  - 支持：功能键（`F1`~`F24`）或带修饰键的组合（例如 `Ctrl+Shift+A`）
  - 若注册失败（被占用/系统限制），会提示失败并保留原快捷键
- 截图后自动保存到文件（`autoSaveToFile`）
  - 开启后会将截图写入 PNG 文件，并记录到截图历史
- 保存目录（`saveDir`）
  - 仅在开启“自动保存”时生效
  - 若未设置，默认使用系统图片目录下的 `ElectronScreenshot` 子目录
- 截图后自动打开编辑器（`openEditorAfterCapture`）
  - 开启后，截图确认保存完成会自动打开编辑器并载入图片
- 遮罩透明度（`maskAlpha`）
  - 范围：`0.60 ~ 0.80`
  - 默认：`0.70`

## 使用流程

1. 启动应用后常驻托盘。
2. 触发截图：
   - 按截图快捷键（默认 `F1`），或
   - 托盘菜单点击“截图”。
3. 按住左键拖拽框选区域。
4. 松开后：
   - 确认：点击 `✓` 或按 `Enter`
   - 取消：按 `Esc` 或右键
5. 确认后：
   - 截图写入剪贴板
   - 按设置决定是否自动保存文件 / 自动打开编辑器

## FAQ

### Windows 多屏混合缩放提示

Windows 在多屏且 DPI 缩放不一致（例如一个 100%，另一个 150%）的环境下，部分机器/驱动可能出现坐标映射不一致，表现为“框选看起来正确，但最终裁剪区域偏移”或截图行为异常。

建议：

- 尽量将多块显示器的缩放比例设置为一致。
- 触发截图时，将鼠标光标放在要截图的那块屏幕上。
- 避免依赖跨屏框选（当前截图流程以单屏为单位）。

### 权限提示

- macOS：需要授予“屏幕录制”权限，否则可能无法抓取屏幕内容。
- Linux（Wayland）：部分桌面环境对截屏能力限制较多，建议使用 X11 或配置可用的 portal。
- Windows：若全局快捷键注册失败，尝试更换快捷键组合，或以管理员权限运行。
