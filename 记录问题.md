## 1. 需求概述

当前截图工具存在以下待优化点：

- 截图快捷键目前设想为 F2，但缺少完整的配置与冲突处理说明
- 应用启动后希望常驻 Windows 系统托盘，但目前托盘区没有图标和菜单
- 截图完成后只能得到原始图片，缺少文字、矩形、箭头等标注编辑能力

本文件用于梳理并明确上述问题，对后续实现提供参考。

## 2. 快捷键 F2 配置与行为

- 默认快捷键：全局截图快捷键默认设置为 F2
- 支持自定义：在应用“设置”中允许用户修改为其他组合键（如 Ctrl+Shift+F2）
- 全局热键行为：
  - 用户在任意窗口按下快捷键时，触发截图功能
  - 尽量不打断当前窗口焦点，仅显示截图界面
- 冲突与失败处理：
  - 注册全局快捷键失败时，弹出提示（可能被其他程序占用）
  - 设置界面中对常见系统快捷键冲突给出警告
- 配置持久化：
  - 用户修改后的快捷键需持久保存（本地配置文件或注册表）
  - 应用重启后自动加载并重新注册该快捷键

## 3. 系统托盘图标与行为

- 启动行为：
  - 应用启动后常驻系统托盘（右下角通知区域）
  - 可在设置中控制是否显示主窗口，默认仅托盘驻留
- 托盘图标：
  - 使用清晰可识别的自定义图标（.ico），适配浅色/深色主题
  - 鼠标悬停时显示提示文字，如“截图助手 - 按 F2 开始截图”
- 托盘右键菜单建议：
  - 开始截图（等同于快捷键触发）
  - 打开主界面/设置
  - 开机自启动（可勾选）
  - 退出应用
- 其他注意点：
  - 退出应用时正确释放托盘图标，避免残留“僵尸图标”
  - 多实例启动时仅保留一个实例及一个托盘图标

## 4. 截图完成后的编辑功能

- 基本流程：
  - 用户通过快捷键或托盘菜单开始截图
  - 框选区域完成截图后，自动打开编辑器窗口显示当前截图
- 编辑工具：
  - 文字工具：点击画布添加文本框，支持输入文字，简要控制字体大小和颜色
  - 矩形/高亮框：绘制实线或半透明矩形，用于框选重点区域，支持简单颜色预设
  - 箭头工具：从起点到终点绘制箭头，用于指示重点，支持线条颜色和粗细预设
- 通用编辑能力：
  - 选中与移动：支持选中已有标注元素并拖动调整位置
  - 删除：支持选中后删除（例如 Delete 键或提供删除按钮）
  - 撤销/重做：可作为第二阶段功能，优先支持 Ctrl+Z / Ctrl+Y
- 导出与关闭：
  - 复制到剪贴板：一键复制当前编辑后的图像，用于直接粘贴到聊天或文档
  - 保存到文件：弹出保存对话框，至少支持 PNG 格式
  - 关闭编辑器时，如有未保存修改，可提示用户是否保存或复制后再关闭

## 5. 典型交互流程示例

- 场景 1：快捷键触发截图
  - 用户在任意窗口按下 F2
  - 进入截图模式并框选区域
  - 截图完成后自动进入编辑器
  - 用户添加矩形和箭头标注
  - 点击“复制到剪贴板”，编辑器关闭
- 场景 2：托盘菜单触发截图
  - 用户右键点击托盘图标，选择“开始截图”
  - 后续流程与快捷键触发的截图一致

## 6. 优先级与迭代建议

- 第一阶段（必须实现）：
  - 实现全局快捷键（默认 F2）触发截图
  - 实现托盘图标与基础右键菜单（开始截图 / 退出）
  - 实现截图后编辑器的基础工具：矩形、箭头、文字 + 复制到剪贴板
- 第二阶段（体验优化）：
  - 完善快捷键自定义与冲突检测
  - 增加更多颜色与粗细预设、撤销重做功能
  - 托盘菜单中增加开机自启动和主界面/设置入口

## 7. 后续实现时的技术细节提示

- 操作系统范围：默认目标平台为 Windows
- 技术栈适配：
  - 结合现有项目（如 WPF/WinForms/Electron/Qt 等）选择合适的全局热键与绘图方案
- 配置存储：
  - 推荐优先使用用户目录下的配置文件，结合需求决定是否使用注册表
- 性能与稳定性：
  - 确保截图与编辑过程对前台应用影响最小，避免明显卡顿
  - 注意内存释放，避免长时间使用后占用过高
